### §title MemoryintheAgeofAIAgents: ASurvey (p.1–2)

This survey paper provides a comprehensive landscape of agent memory research, examining memory through three unified lenses—forms, functions, and dynamics—and proposing clearer taxonomies to address fragmentation in the field.

**Key points:**
- [claim] Memory is a core capability of foundation model-based agents, underpinning long-horizon reasoning, continual adaptation, and effective environmental interaction. (p.1)
- [claim] The field of agent memory has become increasingly fragmented due to differing motivations, implementations, assumptions, evaluation protocols, and loosely defined terminologies. (p.1)
- [claim] Traditional taxonomies such as long/short-term memory are insufficient to capture the diversity and dynamics of contemporary agent memory systems. (p.1)
- [method] The survey distinguishes agent memory from related concepts including LLM memory, retrieval augmented generation (RAG), and context engineering. (p.1)
- [result] Three dominant realizations of agent memory are identified: token-level, parametric, and latent memory. (p.1)
- [method] A finer-grained functional taxonomy is proposed that distinguishes factual, experiential, and working memory, moving beyond coarse temporal categorizations. (p.1)
- [definition] Memory dynamics encompass how memory is formed, evolved, and retrieved over time as agents interact with their environments. (p.1)
- [claim] Emerging research frontiers include automation-oriented memory design, reinforcement learning integration, multimodal memory, shared memory for multi-agent systems, and trustworthiness issues. (p.1)

**Defines:**
- `forms` — The structural realizations of agent memory (token-level, parametric, latent) (p.1)
- `functions` — The functional roles of agent memory (factual, experiential, working) (p.1)
- `dynamics` — The temporal processes of memory formation, evolution, and retrieval (p.1)

###### §engineering Engineering (p.9–13)

This section delineates Agent Memory from three related paradigms—LLM Memory, RAG, and Context Engineering—by examining their overlaps, distinctions, and historical relationships, arguing that Agent Memory uniquely focuses on persistent, self-evolving cognitive state across tasks and interactions.

**Key points:**
- [claim] Agent memory largely subsumes what was historically called LLM memory, as many early 'LLM memory' systems (e.g., MemoryBank, MemGPT) actually addressed agentic challenges like tracking user preferences and accumulating experience across multi-turn interactions. (p.9)
- [definition] LLM-internal memory mechanisms—such as KV cache management, long-context architectural modifications (RWKV, Mamba, diffusion-based LMs), and attention-sparsity methods—are outside the scope of agent memory because they focus on intrinsic model dynamics rather than agentic behavior. (p.9)
- [claim] Few-shot prompting can be viewed as a form of long-term agent memory, where past exemplars or distilled task summaries serve as reusable knowledge incorporated through retrieval or context injection. (p.9)
- [claim] Self-reflection and iterative refinement methods align with short-term, inside-trial agent memory, as the agent leverages intermediate reasoning traces from prior attempts within the same task. (p.9)
- [claim] KV compression and context-window management techniques, when used to preserve salient information across a single task, function as short-term memory mechanisms in an agentic sense. (p.9)
- [definition] Classical RAG augments LLMs with access to static, externally maintained knowledge sources and does not maintain an evolving memory of past interactions, distinguishing it from agent memory. (p.10)
- [definition] Agent memory systems are instantiated within an agent's ongoing interaction with an environment, continuously incorporating new information from the agent's actions and environmental feedback into a persistent memory base. (p.10)
- [caveat] The boundary between RAG and agent memory has become increasingly blurred as retrieval systems grow more dynamic, with systems like HippoRAG being claimed by both communities. (p.10)
- [result] A practical distinction between RAG and agent memory lies in task domain: RAG is evaluated on knowledge-intensive single-inference benchmarks (e.g., HotpotQA), while agent memory is evaluated on multi-turn, temporally dependent, or environment-driven benchmarks (e.g., LoCoMo, LongMemEval, SWE-bench). (p.10)
- [claim] Graph-based retrieval forms a structural backbone for both RAG and agent memory, but only agent memory treats the graph as a living, evolving representation of experience. (p.11)
- [claim] Agentic RAG occupies the closest conceptual space to agent memory, but the key distinction is that agentic RAG typically operates over an external task-specific database, whereas agent memory maintains an internal, persistent, self-evolving memory base that accumulates knowledge across tasks. (p.11)
- [definition] Context engineering and agent memory are best understood as an intersection of distinct operational paradigms rather than a hierarchical subsumption: context engineering treats the context window as a constrained resource to optimize, while agent memory focuses on cognitive modeling of a persistent entity. (p.11)
- [claim] The boundary between context engineering and agent memory dissolves in working memory scenarios during long-horizon interactions, where both rely on the same summarization, dynamic retrieval, and recursive state update mechanisms. (p.12)
- [claim] Rolling summary serves as a shared foundational primitive functioning simultaneously as a buffer management strategy and a transient episodic memory mechanism in both context engineering and agent memory. (p.12)

This section introduces a three-part taxonomy of experiential memory for AI agents—Case-based, Strategy-based, and Skill-based—describing their distinct roles, representative methods, and technical characteristics across carrier, form, and optimization dimensions.

**Key points:**
- [definition] Case-based memory stores minimally processed historical episodes with high fidelity to support direct replay and use as in-context exemplars. (p.38)
- [definition] Strategy-based memory distills transferable reasoning patterns, workflows, and high-level insights from past trajectories to guide planning across diverse scenarios. (p.38)
- [definition] Skill-based memory encapsulates executable procedural capacities—from code snippets to API protocols—that operationalize abstract strategies into verifiable actions. (p.38)
- [claim] Case-based memory offers high informational fidelity and verifiable evidence for imitation, but imposes challenges in retrieval efficiency and context window consumption. (p.39)
- [method] Memento uses soft Q-learning to dynamically refine the probability of selecting high-utility past trajectories for retrieval. (p.39)

- [definition] Strategy-based memory is categorized into three types of increasing abstraction: atomic insights, sequential workflows, and schematic patterns. (p.40)

###### §twotypesof-workingmemory TwoTypesof WorkingMemory (p.43–48)

This section distinguishes two types of working memory for LLMs: single-turn working memory for input condensation and observation abstraction within a single forward pass, and multi-turn working memory for temporal state maintenance across sequential interactions via state consolidation, hierarchical folding, and cognitive planning.

**Key points:**
- [definition] Single-turn working memory actively constructs a writable workspace by filtering and transforming raw information to increase density under fixed attention and memory budgets. (p.43)
- [definition] Input condensation techniques fall into three paradigms: hard condensation (discrete token selection), soft condensation (dense latent vector encoding), and hybrid approaches combining both. (p.43)
- [caveat] Hard condensation methods like LLMLingua and LongLLMLingua use token perplexity to discard predictable or task-irrelevant content, but risk severing syntactic or semantic dependencies. (p.43)
- [caveat] Soft condensation methods such as Gist, ICAE, and AutoCompressors achieve high compression ratios but require additional training and may obscure fine-grained details. (p.43)
- [definition] Observation abstraction transforms raw high-dimensional observations into structured semantic representations to prevent agents from being overwhelmed by raw data. (p.43)
- [caveat] Single-turn working memory optimization is strictly intra-turn, addressing breadth and complexity of static inputs rather than temporal continuity of dynamic interactions. (p.45)
- [claim] In multi-turn settings, the primary bottleneck shifts from instantaneous context capacity to continuous maintenance of task state and historical relevance. (p.45)
- [result] Accumulation of interaction history in long-horizon tasks inevitably saturates attention budgets, increases latency, and induces goal drift. (p.45)
- [definition] State consolidation maps an ever-growing interaction trajectory into a fixed-size state space through dynamic updates, treating interaction as a streaming environment. (p.45)
- [definition] Hierarchical folding decomposes task trajectories based on subgoals, maintaining fine-grained traces only while a subtask is active and folding completed sub-trajectories into concise summaries. (p.45)
- [definition] Cognitive planning elevates working memory to a forward-looking structure that guides future actions, rather than merely summarizing the past. (p.45)
- [result] Multi-turn working memory mechanisms effectively decouple reasoning performance from interaction length by integrating state consolidation, hierarchical folding, and cognitive planning. (p.46)

**Defines:**
- `Single-turn Working Memory` — Working memory operating within a single forward pass to condense and abstract massive immediate inputs such as long documents or multimodal streams (p.43)
- `Multi-turn Working Memory` — Working memory operating across sequential interactions to maintain task states, goals, and constraints through continuous read-execute-update loops (p.43)
- `Hard condensation` — Input condensation paradigm that discretely selects tokens based on importance metrics such as perplexity (p.43)
- `Soft condensation` — Input condensation paradigm that encodes variable-length contexts into dense latent vectors or memory slots (p.43)
- `Hybrid condensation` — Input condensation paradigm combining global semantic adapters (soft) with token-level retention probabilities (hard) (p.43)
- `State consolidation` — Multi-turn working memory strategy that maps growing interaction trajectories into fixed-size state spaces via dynamic updates (p.45)
- `Hierarchical folding` — Multi-turn working memory strategy that decomposes task trajectories by subgoals and folds completed sub-trajectories into concise summaries (p.45)
- `Cognitive planning` — Multi-turn working memory strategy that creates and maintains an externalized plan or world model to guide future agent actions (p.45)
- `PE` — Prompt engineering and inference-time optimization techniques that require no parameter updates (p.43)

###### §fivecategoriesof-memory-format FiveCategoriesof Memory FormationOperations (p.48–55)

This section taxonomizes memory formation operations into five categories—Semantic Summarization, Knowledge Distillation, Structured Construction, Latent Representation, and Parametric Internalization—describing their mechanisms, representative systems, and trade-offs.

**Key points:**
- [definition] Semantic summarization transforms raw observational data into compact, semantically rich summaries capturing global high-level information rather than specific factual details. (p.48)
- [result] Incremental semantic summarization fuses new information chunks with existing summaries over time, circumventing the O(n²) computational burden of full-sequence processing. (p.50)
- [method] Reinforcement learning methods such as Mem1 (PPO) and MemAgent (GRPO) enhance the LLM's own summarization capability rather than relying on auxiliary networks. (p.50)
- [result] Partitioned semantic summarization divides information into distinct semantic partitions and generates separate summaries, offering superior efficiency and finer-grained semantics compared to incremental approaches but risking loss of cross-partition dependencies. (p.50)
- [claim] Semantic summarization is a lossy compression mechanism that prioritizes global semantic coherence over local factual precision, making it ideal for long-term dialogue but limiting utility in evidence-critical tasks. (p.50)
- [definition] Knowledge distillation operates at finer granularity than semantic summarization, extracting reusable factual and experiential knowledge from interaction trajectories or documents. (p.50)
- [definition] Distilling factual memory transforms raw interactions into explicit declarative knowledge about users and environmental states to maintain consistency and adaptability. (p.51)
- [definition] Distilling experiential memory extracts planning strategies from historical trajectories, enabling cross-task knowledge transfer and movement toward lifelong learning. (p.51)
- [method] H2R introduces a two-tier reflection mechanism providing both high-level planning insights and fine-grained step-wise execution insights, addressing the gap in prior work that lacked step-level granularity. (p.51)
- [method] Trainable distillation methods such as Memory-R1 use a dedicated LLM Extract module to obtain experiential and factual knowledge in an end-to-end framework. (p.51)
- [claim] The five memory formation strategies are not mutually exclusive, and a single algorithm can integrate multiple strategies and translate knowledge across different representations. (p.48)
- [definition] Structured construction organizes amorphous source data into explicit topological representations such as knowledge graphs or hierarchical trees to enhance explainability and support multi-hop reasoning. (p.48)
- [definition] Latent representation encodes raw experiences directly into machine-native formats such as vector embeddings or KV states within a continuous latent space. (p.48)
- [definition] Parametric internalization consolidates external memories into the model's weight space through parameter updates, transforming retrievable information into the agent's intrinsic competence. (p.48)

###### §c-forgetting-b-updating c. Forgetting b. Updating (p.56–60)

This section covers three memory evolution mechanisms: consolidation (local, cluster, global), updating (external memory and model editing), and forgetting (time-based, frequency-based, importance-driven), describing how agentic memory systems maintain accuracy, coherence, and efficiency over time.

**Key points:**
- [result] Cluster-level fusion methods like PREMem and EverMemOS capture cross-instance regularities by aligning new memory clusters with existing ones and merging semantically similar entries. (p.56)
- [claim] Global integration consolidates high-level structured knowledge from complete experience history, improving generalization, reasoning accuracy, and personalized decision-making. (p.57)
- [caveat] Consolidation risks information smoothing, where outlier events or unique exceptions may be lost during abstraction, reducing sensitivity to anomalies. (p.57)
- [definition] Memory update aims to maintain factual consistency and continual adaptation without full model retraining by performing localized correction and synchronization. (p.57)
- [result] External memory update mechanisms have evolved from rule-based destructive replacement to temporally aware soft deletion, dual-phase strategies, and reinforcement-learning-based policy learning. (p.57)
- [result] Zep introduced temporal annotations that mark conflicting facts with invalid timestamps rather than deleting them, preserving both semantic consistency and temporal integrity. (p.57)
- [result] Mem-α formulates memory updating as a policy-learning problem, enabling the LLM to learn when and how to update, achieving dynamic trade-offs between stability and freshness. (p.58)
- [definition] Model editing performs direct modifications within the model's parameter space to correct or inject knowledge without full retraining. (p.58)
- [claim] The key challenge in memory updating is the stability-plasticity dilemma: determining when to overwrite existing knowledge versus treating new information as noise. (p.58)
- [definition] Memory forgetting deliberately removes outdated, redundant, or low-value information to free capacity and maintain focus on salient knowledge. (p.58)
- [method] Forgetting mechanisms are categorized into time-based, frequency-based, and importance-driven forgetting corresponding to creation time, retrieval activity, and semantic valuation. (p.58)

- [result] MemEvolve proposes a meta-evolutionary framework that jointly evolves both agents' experiential knowledge and their underlying memory architecture. (p.59)
